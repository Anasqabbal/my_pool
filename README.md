## 📌 Overview
Welcome to **My Pool**, a complete archive of my work during the rigorous 42 Network (1337) Piscine. This intensive 4-week programming bootcamp acts as the ultimate test of resilience, problem-solving, and rapid learning. This repository tracks my progression from basic Unix terminal commands to complex C algorithms and memory management.

As a Software Engineer, the Piscine was my crucible. It taught me not just how to code in C and Shell, but how to read documentation (man pages), collaborate with peers, and debug relentlessly without relying on modern IDEs or standard libraries.

## 💡 Core Concepts Explored
### Foundational C and Shell Scripting
**Definition:** The 42 Piscine covers the absolute fundamentals of programming, including Unix shell navigation, bash scripting, and the C programming language from basic syntax up to complex pointers and data structures.
**Problem Solved:** This bootcamp bridges the gap between zero knowledge and self-sufficient software engineering. It removes all abstractions, forcing students to manipulate raw memory, handle string arrays manually, and write robust, error-free logic using strict coding norms (The 42 Norm).

## 🚀 Features
- **Shell Scripting (`Shell00`, `shell01`):** Unix environment setup, file permissions, pipes, data filtering (`grep`, `awk`), and basic bash scripts.
- **C Fundamentals (`c00` - `c02`):** Variables, loops, conditional statements, basic string manipulation, and standard output (using the `write` system call).
- **Pointers & Logic (`c03` - `c05`):** Pointer arithmetic, memory addressing, custom standard library functions (like `strcmp`, `strcat`), and mathematical algorithms (factorials, prime numbers).
- **Advanced Concepts (`c06` - `c08`):** Command-line argument parsing, dynamic memory allocation (`malloc`, `free`), custom structs, and header files.

## 🛠️ Usage
To test the functions in this repository, navigate into any specific module (e.g., `c04`) and compile the C files using `gcc`:
```bash
cd c04/ex00
# Note: You may need to create a simple main.c file to test the functions
gcc -Wall -Wextra -Werror ft_strlen.c main.c -o test_strlen
./test_strlen
```
*Note: The code in these modules is written strictly as utility functions following the 42 Norm, requiring a driver (`main.c`) to execute.*
